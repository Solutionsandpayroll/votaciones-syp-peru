1 Descripción General del Sistema

Plataforma web desarrollada para la elección del Supervisor de SST, que garantiza el voto individual, secreto e irrepetible de cada trabajador autorizado. El sistema está disponible desde cualquier dispositivo con acceso a internet y no requiere instalación.

3 Funcionamiento del Sistema

El proceso de votación sigue estos pasos:

  1. El trabajador ingresa al sistema y selecciona su nombre de la lista autorizada.
  2. El sistema verifica en tiempo real si esa persona ya votó. Si es así, bloquea el acceso de forma permanente.
  3. El trabajador selecciona su candidato y confirma el voto.
  4. El sistema registra la participación y el conteo, mostrando la pantalla de confirmación.
  5. El acceso queda bloqueado para ese nombre — no es posible votar dos veces.
4 Garantía de Anonimidad del Voto

Esta es la garantía central del sistema. El diseño de la base de datos hace imposible —incluso técnicamente— conocer quién votó por quién.

Estructura de las tablas:

Tabla: votantes
id número
nombre texto
fecha_hora timestamp
Tabla: resultados
candidato_id número
candidato_nombre texto
total_votos contador
⚠ No existe ningún campo ni relación que vincule un nombre con un candidato. Ese dato nunca se escribe en ningún lugar del sistema.
✔ Por qué es imposible la correlación

• La tabla votantes registra únicamente quién participó y cuándo. No contiene ningún campo de candidato.

• La tabla resultados almacena solo un contador que se incrementa en 1. No guarda el nombre del votante ni ningún registro individual del voto.

• Aunque alguien tuviera acceso directo a la base de datos, solo vería: "estas personas votaron" y "este candidato tiene X votos" — sin ninguna conexión entre ambas tablas.

• El sistema funciona como una urna física: el control de asistencia es completamente independiente del conteo de votos.

5 Acceso a Resultados
  • ✔ La página de resultados está protegida con contraseña de administrador.
  • ✔ Los resultados muestran únicamente: votos por candidato, porcentaje, y el historial de quiénes participaron (sin indicar por quién votaron).
  • ✔ Los datos se actualizan en tiempo real (cada 10 segundos).
  • ✔ El administrador puede exportar los resultados a PDF directamente desde la plataforma.
6 Durabilidad de los Datos (5 años)
  • ✔ Los datos están en Neon PostgreSQL, con almacenamiento gratuito permanente vinculado al correo corporativo — no a una suscripción de pago.
  • ✔ No depende de que ninguna persona custodie un archivo ni de herramientas de terceros como Microsoft 365.
  • ✔ Los datos pueden exportarse en cualquier momento desde el panel de Neon como respaldo en formato SQL.
  • ✔ El código fuente está respaldado permanentemente en GitHub.
  • ✔ Si el hosting cambiara en el futuro, el sistema puede redeployarse en minutos desde el mismo repositorio.
7 Seguridad del Sistema
  • ✔ Las credenciales de base de datos nunca están en el código fuente — se gestionan como variables de entorno en el servidor.
  • ✔ La conexión a la base de datos ocurre únicamente en el servidor, nunca en el navegador del usuario.
  • ✔ Toda la comunicación está cifrada con HTTPS.
  • ✔ El sistema valida en el servidor que un votante no haya votado antes de registrar el voto, previniendo duplicados incluso si se manipula el navegador.
  • ✔ La contraseña de administrador está configurada como variable de entorno, no escrita en el código.
